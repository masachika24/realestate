## 1. 情報収集・調査系（Intelligence）

不動産投資判断の基盤となる情報を自動収集・整理するエージェント群。

| エージェント | 役割 | 主要インプット | 期待アウトプット |
|-------------|------|---------------|-----------------|
| **market-researcher** | マクロ指標（人口動態・雇用・金利）＋ミクロ指標（周辺相場・利回り推移）の収集 | 政府統計、日銀データ、不動産指数 | 市場動向レポート（月次）、投資適格エリアランキング |
| **property-scout** | 不動産ポータル（athome, SUUMO, 楽待 etc.）巡回、新着物件の自動検知と通知 | 物件ポータルAPI / スクレイピング | 新着物件アラート、条件マッチ物件リスト |
| **area-analyst** | 対象エリアの将来性評価 — 再開発計画、インフラ整備、学区、犯罪率、ハザードマップ | 都市計画情報、公共データ、自治体公表情報 | エリア評価スコアカード、リスクマップ |
| **transaction-history-researcher** | 登記情報・取引履歴から実勢価格・価格変動パターンを分析 | 登記情報、不動産取引DB | 成約価格推移グラフ、価格予測モデル |
| **off-market-finder** | 業者ネットワーク・SNS・競売情報から非公開物件を発掘 | 業者ネットワーク、SNS、競売情報 | 非公開物件リスト、オフマーケット機会アラート |

## 2. 分析・評価系（Analysis）

収集した情報をもとに定量分析を行い、投資判断の材料を生成するエージェント群。

| エージェント | 役割 | 主要インプット | 期待アウトプット |
|-------------|------|---------------|-----------------|
| **deal-analyzer** | 収支シミュレーション — NOI / Cap Rate / IRR / Cash-on-Cash / DSCR の自動計算 | 物件情報、資金計画 | 投資収支計算書、指標サマリー |
| **risk-assessor** | 空室率予測、金利上昇ストレステスト、法令変更リスク、災害リスクの定量化 | 市場データ、ハザード情報、金利予測 | リスク評価レポート、ストレステスト結果 |
| **comparable-analyzer** | 類似取引事例（comps）との比較評価、適正価格レンジの算出 | 取引事例DB、物件属性 | 比較分析レポート、適正価格レンジ提案 |
| **tax-optimizer** | 減価償却シミュレーション、固定資産税試算、法人/個人の税務最適化 | 税法データ、物件情報、事業スキーム | 税務最適化提案書、減価償却計画 |
| **financing-analyzer** | 金融機関別ローン条件比較、レバレッジ最適化、返済シミュレーション | ローン商品DB、金利情報 | 融資比較表、最適ローン提案 |
| **renovation-estimator** | リノベーション・リフォーム費用の概算見積もりと費用対効果分析 | 工事単価DB、物件状態 | リノベ費用見積書、費用対効果分析 |

## 3. 文書・コミュニケーション系（Documents & Communication）

対外的・対内的なコミュニケーションを効率化するエージェント群。

| エージェント | 役割 | 主要インプット | 期待アウトプット |
|-------------|------|---------------|-----------------|
| **email-reader** | 受信メールの自動分類（問合せ / 契約書類 / 請求書 / 営業 / スパム）＋要約＋優先度付け | 受信メール（IMAP / API） | メール分類結果、優先度付き要約リスト |
| **email-drafter** | オファーレター、価格交渉メール、物件問い合わせ、管理会社への指示文などの下書き作成 | テンプレート、文脈情報 | メール下書き文面、送信承認依頼 |
| **contract-reviewer** | 売買契約書・重要事項説明書・賃貸借契約書の条項チェック、リスク条項のハイライト | 契約書PDF/テキスト | 契約レビューレポート、リスク条項一覧 |
| **report-generator** | 投資提案書、月次運用レポート、年次確定申告用資料の自動生成 | 分析結果、テンプレート | 投資提案書（PDF）、運用レポート（月次/年次） |

## 4. 取引実行・管理系（Transaction Management）

具体的な取引プロセスを管理・支援するエージェント群。

| エージェント | 役割 | 主要インプット | 期待アウトプット |
|-------------|------|---------------|-----------------|
| **offer-strategist** | 売主の状況・市場データから初回オファー価格と上限価格の戦略立案 | 市場データ、売主情報、comps | オファー戦略書、交渉シナリオ |
| **due-diligence-coordinator** | デューデリ項目（物件調査・権利調査・法令調査・収支検証）のチェックリスト管理と進捗追跡 | DD チェックリストテンプレート | DD 進捗ダッシュボード、未完了項目アラート |
| **closing-coordinator** | 決済・引渡し手続きのタイムライン管理、必要書類の期限管理、関係者リマインド | 契約情報、期限データ | クロージングタイムライン、リマインド通知 |
| **negotiation-advisor** | カウンターオファーの分析、過去の交渉データに基づくアドバイス生成 | 交渉履歴、相手方プロファイル | 交渉アドバイスレポート、価格交渉シミュレーション |

複数の分析結果を統合し、高度な意思決定を支援するエージェント群。

| エージェント | 役割 | 主要インプット | 期待アウトプット |
|-------------|------|---------------|-----------------|

## 9. 導入優先度マップ（Phase 1〜4）

全エージェントを一斉に開発することは非現実的であるため、以下の優先度基準に基づき段階的導入を計画する。

### 優先度評価基準
1. **即効性**: 導入後すぐに業務効率化の効果が得られるか
2. **汎用性**: 1つのエージェントがカバーする業務範囲の広さ
3. **実装容易性**: 技術的難易度の低さ（= 早期リリース可能か）
4. **依存関係**: 他のエージェントの前提となるか

### Phase 1：即効性重視（コア業務の効率化）

| エージェント | カテゴリ | 選定理論根拠 |
|-------------|---------|-------------|
| **email-reader** | 文書・コミュニケーション | 即効性が最も高い。日々のメール処理が最大の時間泥棒であり、自動分類＋要約だけで1日1〜2時間の削減が見込める |
| **email-drafter** | 文書・コミュニケーション | email-reader とセットで導入することで、受信→返信のワークフロー全体を効率化。定型的な問い合わせ回答の自動下書きは即戦力となる |
| **deal-analyzer** | 分析・評価 | 投資判断の中核。現在スプレッドシートで手作業している収支計算を自動化することで、判断速度と精度が飛躍的に向上する。また後続エージェント（comparable-analyzer, report-generator 等）の基盤となる |

**Phase 1 合計**: 3 エージェント  
**期待効果**: 日次業務時間 20〜30% 削減、投資判断スピード 2〜3 倍

### Phase 2：情報収集の自動化とリスク管理

| エージェント | カテゴリ | 選定理論根拠 |
|-------------|---------|-------------|
| **property-scout** | 情報収集・調査 | 情報収集の自動化により機会損失を防止。新着物件の見逃しによる逸失利益は計り知れない。deal-analyzer との連携で初期スクリーニングまで自動化可能 |
| **market-researcher** | 情報収集・調査 | マクロ・ミクロの市場データを継続的にモニタリングすることで、投資判断の質を底上げ。property-scout と並行導入することで情報基盤を確立 |
| **contract-reviewer** | 文書・コミュニケーション | 契約書の見落としリスクを低減。特に初回契約時・大口取引時のレビュー工数を大幅削減。法的リスク回避による損失防止効果は極めて大きい |
| **risk-assessor** | 分析・評価 | deal-analyzer の分析結果を補完し、定性・定量両面からのリスク評価を追加。空室率ストレステストや金利変動シミュレーションは投資判断の必須要素 |

**Phase 2 合計**: 4 エージェント（累計 7）  
**期待効果**: 情報収集時間 50% 削減、契約リスク見落とし率 90% 低減

### Phase 3：運用フェーズの効率化

| エージェント | カテゴリ | 選定理論根拠 |
|-------------|---------|-------------|
| **lease-manager** | 運用・管理 | ポートフォリオが拡大すると契約更新管理が指数関数的に複雑化する。更新漏れ・賃料改定機会の逸失を防止し、安定収益を維持するために必須 |
| **maintenance-coordinator** | 運用・管理 | 物件数増加に伴い修繕対応工数が急増する。自動トリアージと業者手配の効率化により、入居者満足度と物件価値の維持を両立 |
| **portfolio-visualizer** | 投資家・資金調達 | 全体最適の可視化により、出口戦略（売却・リファイナンス・追加投資）の検討が可能になる。投資家への説明資料としても活用できる |
| **comparable-analyzer** | 分析・評価 | Phase 2 までのデータ蓄積を活用し、より高精度な比較分析を提供。deal-analyzer の分析精度を一段階引き上げる |
| **rent-collector** | 運用・管理 | 入金管理の自動化により管理工数を削減。滞納の早期検知と自動催促によりキャッシュフローの安定化に寄与 |

**Phase 3 合計**: 5 エージェント（累計 12）  
**期待効果**: 運用管理工数 40〜50% 削減、滞納リスク早期検知率 80% 向上

### Phase 4：高度化・拡張フェーズ

| エージェント | カテゴリ | 選定理論根拠 |
|-------------|---------|-------------|
| **area-analyst** | 情報収集・調査 | Phase 1〜3 の基盤が整った段階で、より深いエリア分析を導入。データ蓄積が進んだ後の方が分析精度が高まる |
| **transaction-history-researcher** | 情報収集・調査 | 登記データ等の大規模データへのアクセスが必要であり、技術的難易度が高いため後回し |
| **off-market-finder** | 情報収集・調査 | 業者ネットワークの構築とデータ整備に時間を要するため、中長期的な取組みとして位置づけ |
| **tax-optimizer** | 分析・評価 | 税法は頻繁に改正されるため、継続的なメンテナンスコストが高い。事業規模が拡大し節税効果が大きくなる段階で導入 |
| **financing-analyzer** | 分析・評価 | 金融機関との関係構築と融資データの蓄積が必要。対象物件数が増加した段階で導入価値が高まる |
| **renovation-estimator** | 分析・評価 | リノベーションの実施頻度が高まる中規模以上のポートフォリオで効果を発揮 |
| **report-generator** | 文書・コミュニケーション | 投資家向け等の定期レポート作成が非効率化してきた段階で導入。Phase 3 以前は手動でも対応可能 |
| **offer-strategist** | 取引実行・管理 | 交渉データの蓄積が前提となるため、ある程度の取引実績が溜まった段階で導入 |
| **due-diligence-coordinator** | 取引実行・管理 | 取引頻度が高まった段階で導入価値が最大化。年数件の取引であれば手動管理で十分 |
| **closing-coordinator** | 取引実行・管理 | 同上。取引量に比例して必要性が高まる |
| **negotiation-advisor** | 取引実行・管理 | 過去の交渉履歴データの蓄積が必要 |
| **tenant-screener** | 運用・管理 | 入居審査は高い精度が求められるため、AI 判断のみに委ねるリスクが大きい。補助ツールとして段階的に導入 |
| **property-performance-tracker** | 運用・管理 | 複数物件の横断的KPI管理が必要になる規模で導入 |
| **vendor-evaluator** | 運用・管理 | 業者データの蓄積に時間を要する |
| **regulation-monitor** | 法務・コンプライアンス | 法改正は頻度が限られるため、定期的手動チェックの代替として導入 |
| **zoning-analyzer** | 法務・コンプライアンス | 建築確認時に必要なため、開発案件が増えた段階で導入 |
| **aml-kyc-checker** | 法務・コンプライアンス | 大口取引時のみ必要。法令要件の厳格化に備えて準備 |
| **investor-reporter** | 投資家・資金調達 | 投資家数が増加した段階で導入 |
| **fundraising-supporter** | 投資家・資金調達 | 資金調達ニーズが高まった段階で導入 |
| **investment-committee-advisor** | 意思決定支援・統合 | 全分析エージェントの出力を統合する性質上、最後に導入 |
| **priority-optimizer** | 意思決定支援・統合 | タスク量・候補物件数の増加に応じて必要性が高まる |
| **scenario-simulator** | 意思決定支援・統合 | risk-assessor の機能を発展させた高度分析として最終段階で導入 |

**Phase 4 合計**: 22 エージェント（累計 34）  
**期待効果**: 全業務領域の AI カバレッジ 80% 以上、意思決定の高度化・定量化
